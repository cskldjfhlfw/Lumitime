# Lumitime 留白微光 UI 改版设计规格

日期：2026-06-25

## 背景

Lumitime 的主身份是个人站，不是企业 SaaS 营销站。工作站只是受邀成员和站主自用的辅助功能，用来提升工作效率。新版 UI 需要美观简约，保持个人表达、内容沉淀和工具可用之间的平衡。

本次范围选择为“首页 + 核心工作流统一”。核心工作流包括工作站、日志提交服务页和提交记录页。API、权限、路由、提交、轮询、重试、筛选等交互逻辑保持不变。

## 目标

- 将首页从偏“入口集合”优化为高质感个人站主页。
- 强化“拾光筑梦 / Lumitime”的黑白微光品牌记忆点。
- 让公开内容入口更清晰：随记、大屏看板、作品、心得。
- 将工作站降级为受邀工具入口，避免抢占首页主叙事。
- 统一工作站、日志提交、记录页的视觉密度、按钮、状态、表单和信息层级。
- 保留现有后端 API 调用、鉴权、CSRF、敏感信息保护和提交记录安全约束。

## 非目标

- 不把 Lumitime 改造成企业 SaaS 或团队协作产品。
- 不新增后端接口，不修改 `lumitimeApi.ts` 的请求语义。
- 不改变登录、注册、管理员权限、受邀成员权限和路由保护逻辑。
- 不改日志提交的请求结构、轮询间隔、重试接口、记录筛选字段。
- 不重构后台管理、大屏看板、内容详情页的业务能力。
- 不引入新 UI 框架或大范围替换 shadcn/Radix 组件体系。

## 视觉方向

采用“留白微光”方向：

- 背景以近白、浅灰、纸感中性色为主，不使用大面积高饱和色。
- 品牌元素保留黑白微光：黑色圆形光点、细边框、局部柔和阴影。
- 页面留白更充分，减少整屏深色压迫感。
- 卡片半径控制在 8px 内，避免过度圆角和装饰性漂浮卡片。
- 文字层级克制：标题清晰，正文行高舒展，工具页面控件文字紧凑明确。
- 图标使用现有 lucide 风格，保持 14px 到 18px 的轻量线性图标。
- 工作站页面仍是工具，但视觉上从“产品主角”调整为“受邀工具架”。

## 信息架构

### 首页

首页第一屏应表达个人站定位：

- 品牌：拾光筑梦 / Lumitime。
- 副文案：Every faint light of time, paves the way for your dream.
- 主说明：用于记录、沉淀和整理个人创作的个人站。
- 主操作：进入随记、查看作品或内容入口。
- 次操作：登录、邀请码注册、查看大屏。

首页模块分区：

- 公开入口：随记、大屏看板。
- 内容沉淀：作品、脚本分享、经验心得。
- 受邀工具：工作站、个人中心、管理后台入口。

访客仍能看到受邀模块的锁定状态。点击锁定模块继续弹出登录/注册引导。

### 工作站

工作站标题从强产品化的“工作站”弱化为“受邀工具”或“工作站工具架”，但路由和功能不变。

页面结构：

- 返回首页/主页的轻量入口。
- 页面标题、说明、服务数量、启用数量、最近记录数量。
- 服务卡片列表。
- 最近提交记录列表。
- 接口加载失败提示和重试按钮。

服务卡片保留状态、摘要、更新时间和进入按钮，但减少视觉噪音，统一边框、阴影、状态徽标和按钮尺寸。

### 日志提交服务页

现有“服务说明 -> 凭证输入 -> 任务参数 -> 关键字段预览 -> 提交按钮 -> 执行追踪 -> 最近记录”的流程保持不变。

UI 调整：

- 顶部服务说明更轻，减少强边框和技术标签视觉占比。
- 表单区使用清晰的步骤分组，但避免厚重卡片堆叠。
- 凭证安全提示保留，并保持“学生学习 App 凭证不是 Lumitime 账号”的表达。
- `ReadonlyDoSavePanel` 保留，只优化为更易扫读的只读数据表。
- 提交按钮、重新填写按钮和状态卡片使用统一尺寸和视觉语言。
- 轮询、`service_request_id`、`request_id`、`polling_url` 信息保持可见。

### 提交记录页

现有筛选、搜索、分页、详情抽屉、重试流程保持不变。

UI 调整：

- 表格区域更像轻量记录台，而不是后台管理表格。
- 安全说明保留，但视觉上降噪。
- 状态徽标、账号掩码、失败码和请求 ID 保持清晰。
- 移动端记录项保留卡片化折叠，桌面端保留表格扫描效率。
- 重试抽屉保持需要重新输入账号和密码，不从历史记录恢复敏感信息。

## 组件与文件边界

预计涉及：

- `frontend/src/pages/HomePage.tsx`
- `frontend/src/layouts/MainNav.tsx`
- `frontend/src/features/home/components/ModuleCard.tsx`
- `frontend/src/pages/WorkstationPage.tsx`
- `frontend/src/features/workstation/components/ServiceCard.tsx`
- `frontend/src/pages/LogSubmitPage.tsx`
- `frontend/src/features/log-submit/components/StatusCard.tsx`
- `frontend/src/pages/RecordsPage.tsx`
- 必要时新增少量首页或布局组件，优先放在对应 feature 目录。

避免修改：

- `frontend/src/shared/api/lumitimeApi.ts`
- `frontend/src/app/App.tsx` 的路由和权限语义。
- 后端文件。
- 测试中覆盖的安全逻辑。

## 状态与交互

必须保持：

- 访客点击锁定模块时弹出登录/注册引导。
- 已登录用户进入脚本、作品、心得和工作站。
- 管理员可进入管理后台。
- 工作站加载服务和最近记录时并行请求。
- 工作站加载失败时可重试。
- 日志提交成功创建服务请求后继续轮询状态。
- 终态后刷新最近记录。
- 记录页支持服务、状态、日期和文本搜索。
- 记录详情可通过 URL 中的 `requestId` 打开。
- 可重试记录必须重新输入学生学习 App 账号和密码。

## 响应式要求

- 首页首屏在移动端改为单列，品牌和主操作优先显示。
- 首页模块在移动端单列或双列，不允许文字溢出按钮或卡片。
- 工作站服务卡片在移动端单列，最近记录可横向信息降级。
- 日志提交页在移动端单列，执行追踪和最近记录下移。
- 记录页桌面端保持表格扫描效率，移动端保持可点选记录项。

## 可访问性与安全

- 所有按钮保留可理解的文字或 `aria-label`。
- 表单输入、日期选择、密码显示按钮保留键盘可用性。
- focus 状态清晰，不依赖动效完成核心操作。
- 密码、DeepSeek Key、完整学生账号不得进入本地持久化或记录展示。
- 不新增会暴露敏感信息的 UI 文案、调试信息或导出入口。

## 验证

实现后需要运行：

- `pnpm typecheck`
- `pnpm test`
- `pnpm build`

浏览器验证：

- 首页访客态：公开入口可进入，锁定入口弹登录提示。
- 首页登录态：受邀入口可进入，个人中心可见。
- 工作站：服务列表、最近记录、接口错误提示和重试按钮。
- 日志提交页：必填校验、提交按钮禁用/启用、创建请求、状态卡片、轮询信息。
- 记录页：筛选、搜索、打开详情、关闭详情、可重试记录重新提交表单。
- 桌面和移动视口无明显重叠、溢出或按钮文字截断。

## 已批准设计决策

- 产品定位选择“个人知识与工具门户”，工作站是附属功能。
- 视觉方向选择“留白微光”。
- 改版范围选择“首页 + 核心工作流统一”。
