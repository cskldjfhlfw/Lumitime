# 上线部署与运维手册: 拾光筑梦 Lumitime

版本：v0.1  
日期：2026-06-17  
适用阶段：测试环境部署 / 生产上线 / 日常运维 / 版本更新 / 故障回滚  
关联文档：概要设计、详细设计、开发计划、测试计划、接口设计、数据模型设计

## 0. 当前实现版部署包

当前代码库已补齐可执行的生产部署包，部署入口位于 `deploy/`：

- `deploy/compose.prod.yml`：编排 `db`、`migrate`、`api`、`nginx`、`backup`、`restore`。
- `deploy/docker/Dockerfile.api`：构建 FastAPI 后端镜像。
- `deploy/docker/Dockerfile.nginx`：构建 React 静态资源并由 Nginx 托管。
- `deploy/nginx/lumitime.conf.template`：HTTPS、HTTP 跳转、SPA 托管和 `/api/` 反向代理模板。
- `deploy/.env.example`：生产环境变量模板，不包含真实密钥。
- `deploy/scripts/backup.sh` 和 `deploy/scripts/restore.sh`：数据库、上传附件和部署配置模板的备份/恢复任务。
- `deploy/README.md`：首次部署、管理员初始化、备份、恢复和更新命令。

当前实现约束：生产环境必须使用 PostgreSQL；首次上线使用空库；管理员通过 `POST /api/v1/auth/bootstrap-admin` 初始化；当前代码没有可独立生产运行的外部 Worker，因此生产环境保持 `LUMITIME_ENABLE_INLINE_WORKER=0`，真实工作站脚本执行待后续接入。

生产推荐命令摘要：

```bash
cp deploy/.env.example deploy/.env
docker compose --env-file deploy/.env -f deploy/compose.prod.yml build
docker compose --env-file deploy/.env -f deploy/compose.prod.yml run --rm migrate
docker compose --env-file deploy/.env -f deploy/compose.prod.yml up -d db api nginx
docker compose --env-file deploy/.env -f deploy/compose.prod.yml run --rm backup
```

## 3. 服务器建议

### 3.1 基础配置

首版个人工作站建议：

| 项目 | 建议 |
| --- | --- |
| CPU | 2 核起 |
| 内存 | 4 GB 起 |
| 磁盘 | 50 GB 起，按附件增长预留 |
| 系统 | Linux 服务器发行版 |
| 网络 | 开放 80 / 443 端口 |

说明：

- 如果作品压缩包较多，应增加磁盘容量或使用对象存储。
- 如果日志自动提交脚本较重，应观察 Worker CPU 和内存占用。

### 3.2 基础目录

建议目录：

```text
/opt/lumitime
├─ app/                 # 项目代码
├─ compose/             # docker compose 文件
├─ data/
│  ├─ db/               # 数据库持久化目录
│  ├─ uploads/          # 上传附件
│  └─ exports/          # 临时导出文件，可按策略清理
├─ backups/
│  ├─ db/               # 数据库备份
│  ├─ uploads/          # 附件备份
│  └─ configs/          # 配置模板备份
└─ logs/                # 宿主机级日志，如需要
```

关键原则：

- 数据库数据不能只存在容器内部。
- 上传附件不能只存在容器内部。
- `.env` 生产文件不能提交到代码仓库。

## 4. Docker Compose 服务

### 4.1 服务组成

```text
reverse-proxy
  └─ HTTPS / 域名 / 请求转发

api
  └─ 前台页面 / 管理后台 / API

nginx
  └─ HTTPS / 域名 / 静态资源 / API 反向代理

worker
  └─ 工作站服务任务执行 / 脚本调用 / 日志脱敏（当前版本未独立部署）

db
  └─ 数据库

backup
  └─ 定时备份数据库、附件、配置模板
```

### 4.2 服务职责

| 服务 | 职责 |
| --- | --- |
| nginx | 处理 HTTPS、域名、反向代理、静态压缩等 |
| api | 提供页面、后台、API、认证、权限、导出 |
| worker | 当前版本未独立部署，后续接入真实脚本时再启用 |
| db | 存储业务数据 |
| backup | 执行每日自动备份和备份清理 |

### 4.3 持久化卷

必须持久化：

- 数据库数据。
- 上传附件。
- 必要的导出文件。
- 备份文件。

不应持久化：

- 容器内部临时缓存。
- 未脱敏原始日志。
- 学生学习 App 密码。

## 5. HTTPS 与域名

### 5.1 HTTPS 要求

Lumitime 必须启用 HTTPS。

原因：

- 用户需要登录 Lumitime 站点账号。
- 日志自动提交服务会输入学生学习 App 账号和密码。
- 管理员后台涉及内容管理、审计和导出。

### 5.2 反向代理职责

反向代理应负责：

- 监听 80 / 443。
- HTTP 自动跳转 HTTPS。
- TLS 证书配置。
- 转发请求到 web 服务。
- 可选：基础限流和请求体大小限制。

### 5.3 证书

可选方案：

- 使用自动证书工具。
- 使用云厂商证书。
- 使用手动上传证书。

运维要求：

- 证书到期前需要自动续期或人工检查。
- HTTPS 配置变更后必须验证登录和服务提交流程。

## 6. 环境变量与配置

### 6.1 配置分类

| 类型 | 示例 | 管理方式 |
| --- | --- | --- |
| 数据库连接 | DB_HOST、DB_USER、DB_PASSWORD | `.env` 或部署平台密钥 |
| Session 密钥 | SESSION_SECRET | `.env`，禁止入库 |
| 文件存储 | STORAGE_PATH 或对象存储配置 | `.env` |
| Worker 配置 | WORKER_INTERVAL、TASK_TIMEOUT | `.env` |
| 域名配置 | APP_URL | `.env` |
| 备份配置 | BACKUP_RETENTION_DAYS | `.env` |

### 6.2 配置安全要求

- `.env` 不提交代码仓库。
- 生产密钥不写入文档正文。
- 不在前端暴露后端密钥。
- 更新前备份配置模板，不备份真实密钥到不安全位置。

### 6.3 配置模板

建议维护：

```text
.env.example
```

其中只写键名和示例说明，不写真实密码。

## 8. 数据库迁移

### 8.1 迁移原则

- 所有结构变更必须通过迁移脚本。
- 更新前必须备份数据库。
- 迁移应尽量向后兼容。
- 高风险迁移必须准备回滚方案。

### 8.2 执行顺序

```text
备份数据库
  ↓
停止或限制写入
  ↓
执行迁移
  ↓
启动新版本服务
  ↓
检查核心链路
```

### 8.3 回滚注意

如果数据库结构已经变更：

- 优先使用兼容性迁移，避免必须立即回滚数据库。
- 如果必须回滚，使用更新前备份恢复。
- 回滚后必须验证数据一致性。

## 9. 备份策略

### 9.1 每日自动备份

每日自动备份内容：

- 数据库。
- 上传附件。
- 配置文件模板。

建议：

- 每日低峰时间执行。
- 备份文件带日期。
- 备份完成后写入备份日志。
- 备份失败应告警或至少在后台提示。

### 9.2 更新前手动备份

每次版本更新前必须手动备份：

```text
数据库
上传附件
配置模板
```

更新前检查：

- 备份文件已生成。
- 备份文件大小合理。
- 可读取备份文件。

### 9.3 备份保留周期

建议：

- 每日备份保留 7 到 30 天。
- 重要版本更新前备份长期保留。
- 备份文件定期清理，避免磁盘占满。

### 9.4 恢复演练

建议定期做恢复演练：

- 在测试环境恢复数据库备份。
- 检查用户、内容、附件、服务记录是否正常。
- 验证登录和工作站服务记录查询。

## 10. 版本更新流程

### 10.1 早期更新方式：Git 构建

适合开发早期：

```text
进入项目目录
  ↓
拉取指定代码版本
  ↓
构建 web 镜像
  ↓
构建 worker 镜像
  ↓
备份数据库和上传附件
  ↓
执行数据库迁移
  ↓
重启 web 和 worker
  ↓
健康检查
  ↓
执行关键链路回归
```

### 10.2 后续更新方式：版本镜像

适合稳定后：

```text
拉取 lumitime-web:vX.Y.Z
拉取 lumitime-worker:vX.Y.Z
  ↓
备份
  ↓
迁移
  ↓
更新 compose 镜像版本
  ↓
重启服务
  ↓
验证
```

### 10.3 更新检查清单

更新前：

- 已查看版本说明。
- 已确认是否有数据库迁移。
- 已完成数据库和附件备份。
- 已确认磁盘空间充足。
- 已确认可以回滚到旧版本。

更新后：

- 主页可访问。
- 登录可用。
- 管理员后台可用。
- 邀请用户可访问内容。
- 工作站服务可创建请求。
- Worker 可处理任务。
- 大屏看板可访问。
- 日志无明显错误。

## 11. 回滚流程

### 11.1 普通回滚

适用于代码或镜像问题，数据库未发生破坏性变更。

```text
停止新版本 web / worker
  ↓
切回上一稳定镜像版本
  ↓
启动 web / worker
  ↓
执行健康检查
  ↓
执行关键链路回归
```

### 11.2 带数据库恢复的回滚

适用于迁移造成不可兼容问题。

```text
停止 web / worker
  ↓
恢复更新前数据库备份
  ↓
恢复必要附件备份
  ↓
切回上一稳定镜像
  ↓
启动服务
  ↓
执行数据一致性检查
  ↓
执行关键链路回归
```

### 11.3 回滚验收

必须验证：

- 用户可登录。
- 内容数据存在。
- 附件可下载。
- 工作站历史记录可查看。
- 大屏看板可访问。
- 管理后台可访问。

## 12. 健康检查与监控

### 12.1 健康检查项

| 服务 | 检查项 |
| --- | --- |
| web | HTTP 健康接口、主页、登录页 |
| worker | 是否定期拉取任务、是否有异常退出 |
| db | 连接是否正常、磁盘空间 |
| reverse-proxy | HTTPS 是否正常、证书是否有效 |
| backup | 最近一次备份是否成功 |

### 12.2 建议健康接口

```text
GET /api/v1/health
```

响应示例：

```json
{
  "code": "OK",
  "message": "healthy",
  "data": {
    "web": "ok",
    "db": "ok",
    "worker": "ok"
  },
  "request_id": "req_health_001"
}
```

### 12.3 日志关注点

重点关注：

- 登录失败频率异常。
- 留言提交频率异常。
- 服务请求失败率异常。
- Worker 异常退出。
- 备份失败。
- 磁盘空间不足。
- HTTPS 证书即将过期。

禁止：

- 记录学生学习 App 密码。
- 记录完整学生学习 App 账号。
- 记录 Cookie、Token、Authorization。

## 14. 故障处理

### 14.1 无法访问网站

检查顺序：

1. 域名解析是否正常。
2. HTTPS 证书是否有效。
3. reverse-proxy 是否运行。
4. web 容器是否运行。
5. 服务器安全组是否开放 80 / 443。

### 14.2 无法登录

检查顺序：

1. 数据库是否连接正常。
2. 用户是否被禁用。
3. Session 配置是否正确。
4. Cookie 是否被浏览器拦截。
5. 后端认证日志是否有异常。

### 14.3 工作站服务一直执行中

检查顺序：

1. worker 容器是否运行。
2. `service_requests` 是否存在 pending / running 任务。
3. Worker 是否能读取任务。
4. 脚本是否卡住。
5. 任务超时配置是否生效。

### 14.4 执行日志为空

检查顺序：

1. Worker 是否保存日志。
2. 脱敏模块是否误删所有内容。
3. 脚本是否没有输出。
4. 当前用户是否有权限查看对应日志。

### 14.5 CSV 导出失败

检查顺序：

1. 管理员权限是否有效。
2. 查询条件是否过大。
3. 导出目录是否可写。
4. 磁盘空间是否充足。
5. 是否存在特殊字符导致 CSV 生成失败。

### 14.6 磁盘空间不足

处理顺序：

1. 检查备份目录大小。
2. 清理过期备份。
3. 清理过期导出文件。
4. 检查上传附件增长。
5. 扩容磁盘或迁移附件到对象存储。
