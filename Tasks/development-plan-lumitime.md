# 开发任务拆分 / 里程碑计划: 拾光筑梦 Lumitime

版本：v0.1  
日期：2026-06-12  
适用阶段：项目排期 / 开发执行 / 进度跟踪 / 验收管理  
关联文档：PRD、概要设计、详细设计、接口设计、数据模型设计、测试计划

## 1. 计划目标

本计划将 Lumitime 首版需求拆分为可执行的开发阶段、模块任务和验收门槛。

核心目标：

- 采用分阶段开发，先 MVP，再补充增强功能。
- MVP 覆盖登录、主页、权限、内容展示、后台内容管理、留言、大屏、工作站服务和模拟日志自动提交。
- 日志自动提交首版使用模拟脚本，验证异步执行、记录、状态、脱敏和重试。
- 真实脚本接入作为后续专项任务。
- 登录页主视觉和反转动画进入一期；小猫交互放入二期。
- 先部署测试环境，功能稳定后部署生产。
- 每个里程碑都有明确验收标准。

## 2. 开发策略

| 项目 | 决策 |
| --- | --- |
| 开发方式 | 分阶段开发 |
| MVP 范围 | 登录、主页、权限、内容展示、后台管理、留言、大屏、工作站服务、模拟日志自动提交 |
| 日志自动提交 | 首版模拟脚本，真实脚本后续接入 |
| 团队假设 | 不绑定人员配置，按模块拆任务 |
| 排期粒度 | 按周里程碑 |
| UI 动效 | 登录页主视觉和反转动画一期；小猫交互二期 |
| 上线策略 | 先测试环境，后生产环境 |
| 验收方式 | 每个里程碑设置验收门槛 |

## 3. 版本分期

### V0.1 MVP

目标：跑通完整首版闭环。

包含：

- 基础项目框架。
- 登录、退出、邀请码注册。
- 三类角色权限。
- 公开主页。
- 脚本、作品、博客前台展示。
- 管理后台基础 CRUD。
- 留言板。
- 工作站服务列表。
- 日志自动提交模拟服务。
- 服务提交记录。
- 日志脱敏。
- 公开大屏看板。
- CSV 导出。
- 测试环境部署。

### V0.2 体验增强

目标：补充视觉动效、交互细节和运维体验。

包含：

- 登录页主视觉优化。
- 登录成功反转动画完善。
- 小猫交互。
- 大屏全屏展示优化。
- 后台筛选、空状态、错误状态完善。
- 导出体验优化。

### V0.3 真实服务接入

目标：接入真实日志自动提交脚本。

包含：

- 真实脚本接入。
- 学校系统异常映射。
- 脚本版本管理。
- 真实执行日志脱敏验证。
- 联调测试。

## 4. 周度里程碑

## Week 1: 项目基础与认证权限

### 目标

建立基础工程、数据模型、认证会话、邀请码注册和角色权限。

### 任务

- T1.1 初始化前后端项目结构。
- T1.2 配置基础路由和页面布局。
- T1.3 建立数据库连接和迁移机制。
- T1.4 创建用户、邀请码、邀请码使用记录表。
- T1.5 实现 Lumitime 登录、退出、当前用户接口。
- T1.6 实现邀请码注册接口。
- T1.7 实现统一鉴权中间件。
- T1.8 实现管理员、邀请用户、访客的基础权限控制。
- T1.9 创建基础审计记录能力。
- T1.10 准备测试账号和测试邀请码。

### 验收标准

- 管理员可以登录并进入 `/admin`。
- 邀请用户可以通过邀请码注册并登录。
- 访客访问后台返回无权限或跳转登录。
- 邀请用户访问后台返回无权限。
- 登录失败不暴露账号是否存在。
- 密码只保存哈希。
- 认证相关 P0/P1 测试用例通过。

## Week 2: 公开页面、主页与内容展示

### 目标

完成公开主页、登录页基础视觉、脚本/作品/博客前台展示和公开留言。

### 任务

- T2.1 实现公开主页。
- T2.2 实现登录页 1:2 布局。
- T2.3 实现登录页黑白主视觉初版。
- T2.4 实现标题和副标题的艺术字体方案。
- T2.5 实现访客看到锁定状态的邀请模块。
- T2.6 创建统一内容表 `content_items`。
- T2.7 实现脚本列表和详情页。
- T2.8 实现作品列表和详情页。
- T2.9 实现博客列表和详情页。
- T2.10 实现随记 / 留言板公开列表。
- T2.11 实现访客留言提交和基础防刷。

### 验收标准

- 主页对访客公开。
- 访客可见随记和大屏入口。
- 访客看到脚本、作品、博客、工作站锁定状态。
- 邀请用户可查看已发布脚本、作品、博客。
- 脚本详情可复制代码。
- 脚本不提供文件下载。
- 访客可提交留言且立即公开。
- 留言频率限制生效。
- Week 2 相关功能和权限测试通过。

## Week 3: 管理后台与内容管理

### 目标

实现管理员后台，支持邀请码、用户、脚本、作品、博客、留言基础管理。

### 任务

- T3.1 实现后台独立布局和侧边栏。
- T3.2 实现后台总览页。
- T3.3 实现邀请码管理。
- T3.4 实现用户管理和禁用用户。
- T3.5 实现脚本管理 CRUD。
- T3.6 实现作品管理 CRUD。
- T3.7 实现作品压缩包上传接口和下载权限配置。
- T3.8 实现博客管理 CRUD。
- T3.9 实现留言隐藏、恢复、删除。
- T3.10 为管理操作写入审计记录。

### 验收标准

- 管理员可创建、禁用、查看邀请码。
- 管理员可禁用用户。
- 管理员可新增、编辑、发布、下架、删除脚本、作品、博客。
- 作品附件 `allow_download=false` 时邀请用户不可下载。
- 邀请用户不能访问任何管理接口。
- 管理后台关键操作有审计记录。
- 后台 P0/P1 测试用例通过。

## Week 4: 工作站服务框架与模拟日志自动提交

### 目标

完成工作站服务框架、异步任务、Worker、模拟脚本、服务提交记录、日志脱敏。

### 任务

- T4.1 创建工作站服务表。
- T4.2 实现工作站服务列表和服务详情页。
- T4.3 创建服务请求表、执行日志表、服务审计表。
- T4.4 实现发起服务请求接口。
- T4.5 实现服务请求状态查询接口。
- T4.6 实现前端轮询状态卡片。
- T4.7 实现 Worker 串行执行框架。
- T4.8 实现 `ScriptRunner` 抽象。
- T4.9 实现模拟日志自动提交脚本。
- T4.10 实现成功、账号密码错误、网络异常、学校系统异常、脚本异常、超时等模拟结果。
- T4.11 实现日志脱敏模块。
- T4.12 实现我的提交记录和详情页。
- T4.13 实现失败重试，且必须重新输入账号密码。

### 验收标准

- 邀请用户可进入工作站并看到服务卡片。
- 访客不能访问工作站。
- 发起服务请求后立即返回 `service_request_id`。

- 模拟脚本成功、失败、超时场景均可验证。
- 邀请用户只能查看自己的提交记录。
- 学生学习 App 密码不入库、不进日志、不出现在响应中。
- 执行日志先脱敏再存储。
- 重试失败记录必须重新输入账号密码。
- 工作站服务 P0 测试用例全部通过。

```text
认证权限
  ↓
内容展示 / 后台管理
  ↓
工作站服务框架
  ↓
模拟日志自动提交
  ↓
服务记录 / 日志脱敏
  ↓

  ↓
```

关键依赖：

- 工作站服务依赖认证权限。

- CSV 导出依赖数据模型和权限控制。
- 安全测试依赖日志脱敏和导出白名单完成。

## 7. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 学生学习 App 密码误入库或日志 | 高危安全问题 | 统一脱敏模块；测试覆盖数据库、日志、导出 |
| Worker 与 Web 分离后临时凭证传递复杂 | 影响工作站服务 | 首版使用安全临时机制；真实脚本接入前专项评审 |
| 登录页动效影响性能 | 影响登录体验 | 支持降级和跳过动画 |
| 留言立即公开导致垃圾内容 | 影响公开页面质量 | 频率限制、长度限制、后续可加敏感词 |
| CSV 导出数据量增长 | 下载慢或超时 | 首版同步，后续升级异步导出 |
| Docker 更新误删数据 | 数据丢失 | 数据库和附件使用持久化卷；更新前备份 |
| 真实学校系统变更 | 自动提交失败 | 脚本版本管理、失败分类、服务启停开关 |

## 8. 最小上线标准

- 管理员、邀请用户、访客权限边界正确。
- 登录、邀请码注册、退出正常。
- 脚本、作品、博客可由管理员管理并由邀请用户查看。
- 留言板可公开查看和留言。
- 工作站服务可使用模拟脚本完成异步执行闭环。
- 服务提交记录和重试流程正常。
- 学生学习 App 密码不入库、不进日志、不导出、不在响应中出现。
- 大屏看板公开可见且只展示聚合数据。
- 管理员 CSV 导出不包含敏感字段。
- 测试环境部署通过。
- P0 缺陷清零。
