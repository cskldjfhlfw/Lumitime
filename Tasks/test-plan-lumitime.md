# 测试计划: 拾光筑梦 Lumitime

版本：v0.1  
日期：2026-06-12  
适用阶段：测试设计 / 开发验收 / UAT / 上线前回归  
关联文档：PRD、权限矩阵、信息架构、接口设计、数据模型设计、视觉交互设计

## 1. 测试目标

本测试计划用于验证 Lumitime 首版是否满足产品、权限、接口、隐私安全和视觉交互要求。

重点目标：

- 验证访客、邀请用户、管理员三类角色的访问边界。
- 验证登录、邀请码注册、内容访问、留言、工作站服务、大屏、后台管理等核心链路。
- 验证学生学习 App 账号密码不会被泄露、保存、展示、导出或写入日志。
- 验证工作站服务异步执行、状态轮询、失败分类、重试和提交记录。
- 验证登录页黑白主视觉、反转动画、小猫交互不会影响核心可用性。
- 验证大屏看板只展示公开聚合数据。
- 验证桌面 Chrome / Edge 和移动端基础适配。

## 2. 测试范围

### 2.1 In Scope

- 认证与邀请码注册。
- 主页和公开页面。
- 脚本分享。
- 个人作品与压缩包下载权限。
- 经验心得 / 技术博客。
- 随记 / 留言板。
- 工作站服务列表。
- 日志自动提交服务。
- 我的提交记录。
- 管理后台。
- 服务提交记录和脱敏执行日志。
- 审计记录。
- 大屏看板。
- 统计记录和服务记录 CSV 导出。
- 视觉交互与动效验收。
- 权限测试。
- 接口测试。
- 隐私安全测试。
- 兼容性测试。
- 基础性能测试。
- 异常恢复测试。

### 2.2 Out Of Scope

- 真实学校系统联调，需等日志自动提交脚本正式接入后补充。
- 大规模压测。
- 第三方 OAuth 登录。
- 移动端原生 App 测试。
- 企业级多管理员角色测试，因为首版管理员不细分。

| 异常恢复测试 | 验证脚本失败、网络异常、超时、接口错误等场景 |

## 6. 权限测试策略

权限测试采用两层验证：

1. 总权限矩阵验证：确认角色在模块级别的可见性和可访问性。
2. 模块内权限验证：确认每个功能内部不会出现越权操作。

关键验证点：

- 访客不能访问脚本、作品、博客、工作站和后台。
- 访客可以访问主页、随记、大屏看板，并可提交留言。
- 邀请用户可以查看脚本、作品、博客、工作站服务。
- 邀请用户不能新增、编辑、删除、发布、下架任何内容。
- 邀请用户只能查看自己的服务提交记录。
- 管理员可以管理全部内容和查看全部脱敏服务记录。
- 审计记录仅管理员可见。

## 7. 隐私安全测试策略

隐私安全测试必须覆盖前端、接口响应、数据库、应用日志、执行日志、导出文件。

重点断言：

- 学生学习 App 密码不入库。
- 学生学习 App 密码不出现在接口响应中。
- 学生学习 App 密码不出现在应用日志中。
- 学生学习 App 密码不出现在服务执行日志中。
- 学生学习 App 密码不出现在 CSV 导出中。
- 学生学习 App 完整账号不入库。
- 学生学习 App 账号只保存掩码和不可逆哈希。
- Cookie、Token、Authorization 请求头不出现在脱敏日志和导出文件中。

## 8. 工作站服务测试策略

首版使用模拟脚本测试异步服务流程。

基本流程：

```text
用户提交服务请求
  ↓
接口立即返回 service_request_id
  ↓
前端轮询请求详情
  ↓
状态从 pending / running 进入 success / failed / timeout
  ↓
用户查看结果摘要
  ↓
失败时可重新提交，但必须重新输入账号密码
```

真实脚本接入后需补充：

- 学校系统登录成功场景。
- 学校系统账号密码错误场景。
- 学校系统验证码或风控场景。
- 学校系统维护或接口变更场景。
- 脚本版本兼容性测试。

## 9. 兼容性范围

首版覆盖：

- 桌面 Chrome 最新稳定版。
- 桌面 Edge 最新稳定版。
- 移动端基础适配，重点验证登录页、主页、工作站服务页、留言页。

不作为首版强制范围：

- Safari 全量测试。
- Firefox 全量测试。
- 低版本浏览器。

## 10. 性能与稳定性目标

建议目标：

| 指标 | 目标 |
| --- | --- |
| 主页首屏 | 常规网络下 3 秒内可交互 |
| 登录接口 | 1 秒内响应，异常时有明确提示 |
| 服务请求创建 | 2 秒内返回 service_request_id |
| 服务轮询 | 默认 2 秒一次，不造成明显页面卡顿 |
| 大屏看板 | 连续展示 30 分钟无明显内存增长或卡死 |
| 后台列表 | 1000 条数据分页查询可用 |

## 11. 进入与退出标准

### 11.1 进入测试标准

- 需求文档已冻结到当前版本。
- 测试环境可访问。
- 管理员和邀请用户测试账号已准备。
- 基础测试数据已准备。
- 模拟日志自动提交脚本已可返回多种状态。
- 接口文档与前端路由基本稳定。

### 11.2 退出测试标准

- P0 用例全部通过。
- P1 用例通过率不低于 95%，未通过项有明确修复计划。
- 无未关闭的高危权限漏洞。
- 无学生学习 App 密码泄露问题。
- 无管理员以外角色访问审计记录或后台管理的漏洞。
- 工作站服务成功、失败、超时、重试主链路通过。
- 上线前回归通过。

## 12. 缺陷优先级

| 优先级 | 定义 | 示例 |
| --- | --- | --- |
| P0 | 阻塞上线或严重安全问题 | 学生 App 密码入库、访客访问后台、邀请用户查看他人提交记录 |
| P1 | 核心功能不可用 | 无法登录、无法提交服务、后台无法发布内容 |
| P2 | 重要体验或边界问题 | 错误提示不清晰、移动端布局错位、导出字段缺失 |
| P3 | 低风险优化 | 文案不统一、动效不够顺滑 |
