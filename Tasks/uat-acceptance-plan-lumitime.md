# UAT 用户验收方案: 拾光筑梦 Lumitime

版本：v0.1  
日期：2026-06-12  
适用阶段：测试环境验收 / 上线前验收 / 受邀用户试用 / 生产发布决策  
关联文档：PRD、权限矩阵、视觉交互设计、测试计划、开发计划、运维手册

## 1. UAT 目标

UAT 是用户验收测试，不替代技术测试。它重点验证 Lumitime 是否符合真实使用预期：

- 访客能理解站点并完成公开留言。
- 邀请用户能顺利访问脚本、作品、博客和工作站服务。
- 管理员能完成日常内容维护、服务管理和记录查询。
- 日志自动提交服务的体验、反馈和隐私提示符合预期。
- 大屏看板展示公开聚合数据，适合作为站点展示入口。
- 登录页视觉和动效传达“拾光筑梦”的品牌寓意，但不影响可用性。

## 2. 验收角色

| 角色 | 验收重点 |
| --- | --- |
| 站点拥有者 / 管理员 | 内容管理、用户管理、服务记录、审计、导出、上线可控性 |
| 邀请用户 | 登录、访问内容、使用工作站服务、查看自己的提交记录 |
| 访客 | 浏览主页、查看公开留言和大屏、提交留言 |
| 技术支持人员 | 部署、备份、回滚、日志、接口和异常处理 |

## 3. UAT 进入标准

开始 UAT 前应满足：

- 测试环境已部署完成。
- HTTPS 已启用。
- 管理员账号已创建。
- 至少 2 个邀请用户账号可用。
- 公开主页、登录页、注册页、留言板、大屏可访问。
- 脚本、作品、博客已有测试数据。
- 工作站服务使用模拟日志自动提交脚本。
- 服务提交记录、日志脱敏、失败重试已完成。
- P0 技术测试用例已通过。

## 4. UAT 退出标准

满足以下条件后可以认为通过 UAT：

- 管理员验收场景全部通过。
- 邀请用户核心场景全部通过。
- 访客核心场景全部通过。
- 无 P0 问题。
- P1 问题已有修复或明确上线前处理计划。
- 未发现学生学习 App 密码泄露。
- 未发现角色越权。
- 站点拥有者确认视觉、文案和主流程符合预期。

## 5. 验收环境与数据

### 5.1 账号

| 类型 | 数量 | 用途 |
| --- | --- | --- |
| 管理员 | 1 | 后台管理验收 |
| 邀请用户 A | 1 | 正常访问和服务提交 |
| 邀请用户 B | 1 | 验证不能查看 A 的提交记录 |
| 访客 | 不登录 | 验证公开访问 |

### 5.2 内容

应准备：

- 至少 2 条脚本，例如“超级粘贴板”“一键清理 D 盘”。
- 至少 2 个作品，其中一个允许下载压缩包，一个不允许下载。
- 至少 2 篇技术博客。
- 至少 3 条留言。
- 至少 1 个工作站服务：日志自动提交。
- 模拟服务结果：成功、账号密码错误、网络异常、脚本异常、超时。

## 6. 访客验收场景

### UAT-VISITOR-001: 浏览公开主页

前置条件：未登录。  
步骤：

1. 打开 Lumitime 主页。
2. 查看品牌名称、品牌句和模块入口。
3. 点击脚本、作品、博客、工作站入口。

验收标准：

- 主页可以直接访问。
- 能理解这是“拾光筑梦 Lumitime”个人工作站。
- 随记和大屏入口可进入。
- 脚本、作品、博客、工作站显示为需要登录或锁定状态。

### UAT-VISITOR-002: 提交留言

步骤：

1. 进入随记 / 留言板。
2. 查看公开留言。
3. 输入昵称和留言内容。
4. 提交留言。

验收标准：

- 访客能看到公开留言。
- 留言提交成功。
- 留言提交后立即公开。
- 留言内容过长或为空时有明确提示。

### UAT-VISITOR-003: 查看大屏看板

步骤：

1. 进入大屏看板。
2. 查看指标和趋势。

验收标准：

- 大屏可以公开访问。
- 展示用户数、开发者数、访问数、作品数、脚本数、博客数、随记数、工作站服务数。
- 不展示用户列表、账号、IP、留言原文、服务日志。

## 7. 邀请用户验收场景

### UAT-INVITED-001: 邀请码注册与登录

步骤：

1. 使用有效邀请码注册。
2. 使用注册账号登录。
3. 登录后进入主页。

验收标准：

- 邀请码注册成功。
- 登录成功。
- 登录后能看到脚本、作品、博客、工作站入口。

### UAT-INVITED-002: 查看并复制脚本

步骤：

1. 进入脚本分享。
2. 打开脚本详情。
3. 点击复制代码。

验收标准：

- 能看到脚本列表和详情。
- 能看到代码和说明。
- 能复制代码。
- 不出现脚本文件下载入口。

### UAT-INVITED-003: 查看作品并验证下载权限

步骤：

1. 进入个人作品。
2. 打开允许下载的作品。
3. 下载压缩包。
4. 打开不允许下载的作品。
5. 尝试查找下载入口。

验收标准：

- 允许下载的作品可以下载。
- 不允许下载的作品不显示下载入口或下载被拒绝。
- 作品内容对访客不可见。

### UAT-INVITED-004: 阅读技术博客

步骤：

1. 进入经验心得。
2. 打开博客详情。

验收标准：

- 能看到博客列表。
- 能阅读已发布博客正文。

### UAT-INVITED-005: 使用日志自动提交模拟服务

步骤：

1. 进入工作站。
2. 打开日志自动提交服务。
3. 输入模拟学生学习 App 账号和密码。
4. 点击自动提交。
5. 等待状态更新。

验收标准：

- 工作站以服务卡片展示。
- 服务页说明清楚。
- 页面提示学生学习 App 账号密码不同于 Lumitime 登录账号密码。
- 提交后立即出现执行中状态。
- 最终显示成功或失败摘要。
- 页面不展示学生学习 App 密码。

### UAT-INVITED-006: 查看自己的提交记录

步骤：

1. 进入我的提交记录。
2. 打开刚才的提交详情。

验收标准：

- 只看到自己的记录。
- 记录包含服务名称、提交时间、状态、service_request_id、耗时、结果摘要、账号掩码。
- 不显示完整执行日志。
- 不显示完整学生账号。
- 不显示学生密码。

### UAT-INVITED-007: 失败后重新提交

步骤：

1. 使用模拟失败场景提交。
2. 打开失败记录。
3. 点击重新提交。
4. 不输入密码尝试提交。
5. 输入账号密码后重新提交。

验收标准：

- 失败记录可以重试。
- 不输入密码不能重试。
- 系统不会自动复用历史密码。
- 重新提交生成新的 service_request_id。

## 8. 管理员验收场景

### UAT-ADMIN-001: 管理邀请码

步骤：

1. 登录管理员账号。
2. 进入邀请码管理。
3. 创建邀请码。
4. 禁用邀请码。
5. 查看邀请码使用记录。

验收标准：

- 管理员能完成邀请码创建、禁用、查看。
- 邀请码使用记录可追溯。

### UAT-ADMIN-002: 管理内容

步骤：

1. 新增脚本并发布。
2. 新增作品并上传压缩包。
3. 配置作品压缩包允许 / 不允许下载。
4. 新增博客并发布。
5. 下架一条内容。

验收标准：

- 管理员能完成内容新增、编辑、发布、下架。
- 前台邀请用户能看到已发布内容。
- 下架内容前台不可见。
- 作品下载权限生效。

### UAT-ADMIN-003: 管理留言

步骤：

1. 进入留言管理。
2. 隐藏一条留言。
3. 恢复该留言。
4. 删除一条留言。

验收标准：

- 隐藏后前台不可见。
- 恢复后前台可见。
- 删除后按设计不再展示。

### UAT-ADMIN-004: 管理工作站服务

步骤：

1. 进入工作站服务管理。
2. 查看日志自动提交服务。
3. 编辑服务说明。
4. 停用服务。
5. 重新启用服务。

验收标准：

- 管理员能编辑服务说明。
- 停用后邀请用户不能发起新请求。
- 启用后邀请用户可再次使用。

### UAT-ADMIN-005: 查看服务记录和脱敏日志

步骤：

1. 进入服务提交记录。
2. 按 service_request_id 搜索记录。
3. 打开详情。
4. 查看完整脱敏执行日志。

验收标准：

- 管理员可查看全部用户的服务记录。
- 可按 service_request_id 检索。
- 完整日志已脱敏。
- 不出现学生密码、完整账号、Cookie、Token、Authorization。

### UAT-ADMIN-006: 导出 CSV

步骤：

1. 导出统计记录 CSV。
2. 导出服务提交记录 CSV。
3. 打开 CSV 检查字段。

验收标准：

- CSV 下载成功。
- 统计 CSV 只包含聚合指标。
- 服务记录 CSV 只包含非敏感字段和脱敏日志。
- 不包含学生密码、完整账号、Token、Cookie、Authorization。

### UAT-ADMIN-007: 查看审计记录

步骤：

1. 进入审计记录。
2. 查看登录、邀请码、服务请求、导出等记录。

验收标准：

- 管理员可查看审计记录。
- 邀请用户和访客不能访问审计记录。
- 审计记录不包含敏感明文。

## 9. 视觉交互验收

### UAT-UI-001: 登录页视觉

验收标准：

- 登录页左右空间接近 1:2。
- 主色为黑白灰。
- 左侧有光束和人物下坠 / 向下爬的视觉意象。
- 标题和副标题有毛笔书写艺术感。
- 登录表单清晰可用。

### UAT-UI-002: 登录成功反转动画

验收标准：

- 登录成功后有从向下到向上的反转动效或可接受的降级效果。
- 动效不影响登录效率。
- 移动端或低性能场景可降级。

### UAT-UI-003: 大屏看板视觉

验收标准：

- 大屏为深色全屏风格。
- 数字指标清楚。
- 趋势变化可读。
- 长时间展示不明显卡顿。

## 10. 权限验收重点

必须通过：

- 访客不能访问脚本、作品、博客、工作站、后台。
- 邀请用户不能访问后台。
- 邀请用户不能新增、编辑、删除、发布、下架内容。
- 邀请用户不能查看他人提交记录。
- 只有管理员能查看审计记录。
- 只有管理员能导出 CSV。

## 11. 隐私验收重点

必须通过：

- 学生学习 App 密码不在页面展示。
- 学生学习 App 密码不在提交记录展示。
- 学生学习 App 密码不在管理员日志展示。
- 学生学习 App 密码不在 CSV 导出。
- 学生完整账号不展示，只展示掩码。
- 服务执行日志展示前已脱敏。

## 12. UAT 问题记录模板

| 字段 | 说明 |
| --- | --- |
| 问题 ID | UAT-ISSUE-001 |
| 发现人 | 谁发现的问题 |
| 角色 | 访客 / 邀请用户 / 管理员 |
| 页面 / 模块 | 出现问题的位置 |
| 操作步骤 | 如何复现 |
| 实际结果 | 实际看到什么 |
| 期望结果 | 应该是什么 |
| 严重程度 | P0 / P1 / P2 / P3 |
| 截图 / 记录 | 可选 |
| 处理状态 | Open / Fixed / Deferred / Closed |

## 13. UAT 签收建议

上线前建议确认：

- 管理员验收人签收。
- 至少 1 名邀请用户试用通过。
- 访客公开访问流程通过。
- 隐私安全检查通过。
- 上线部署检查清单通过。

签收结论：

| 结论 | 含义 |
| --- | --- |
| 通过 | 可进入生产发布 |
| 有条件通过 | 存在非阻塞问题，可上线后修复 |
| 不通过 | 存在 P0 / 关键 P1 问题，不能上线 |
