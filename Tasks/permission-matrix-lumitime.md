# 权限矩阵: 拾光筑梦 Lumitime 个人工作站

版本：v0.1  
日期：2026-06-12  
适用阶段：产品设计 / 技术评审 / 测试用例设计  
关联文档：PRD: 拾光筑梦 Lumitime 个人工作站

## 1. 权限设计结论

Lumitime 是个人工作站，不是多人内容发布平台。首版权限设计以“个人展示、邀请访问、有限互动、管理员统一维护”为原则。

首版仅设置 3 类角色：

| 角色 | 定义 | 权限定位 |
| --- | --- | --- |
| 管理员 | 站点拥有者，首版唯一管理角色 | 拥有全部内容、用户、工作站服务、留言、统计、审计的管理权限 |
| 邀请用户 | 通过邀请码注册并登录的用户 | 查看邀请内容，使用工作站服务，复制脚本代码，下载被允许的作品附件，查看自己的提交记录 |
| 访客 | 未登录用户 | 查看公开主页、公开留言板、公开大屏看板，提交留言 |

首版管理员不细分。未来如有需要，可扩展为内容管理员、工作站管理员、审计管理员等角色，但不进入首版范围。

## 2. 总体权限规则

- GR-1: 管理员是唯一可以新增、编辑、删除、发布、下架脚本、作品、博客和工作站服务的角色。
- GR-2: 邀请用户和访客都不能发布、编辑、删除任何站内内容。
- GR-3: 脚本、个人作品、经验心得、工作站仅对邀请用户和管理员可见。
- GR-4: 主页、随记 / 留言板、大屏看板对访客公开。
- GR-5: 脚本分享只展示代码文本和说明，不提供脚本文件下载。
- GR-6: 脚本代码允许邀请用户复制，不尝试做不可可靠执行的“禁止复制”限制。
- GR-7: 个人作品的压缩包下载权限由管理员按单个作品配置。
- GR-8: 工作站本质是服务集合，例如“实习日志自动提交”是其中一个服务。
- GR-9: 首版所有邀请用户都能看到所有工作站服务，后续预留单服务授权能力。
- GR-10: 邀请用户可以查看自己的服务提交记录和完整脱敏执行日志。
- GR-11: 管理员可以查看全部服务提交记录和完整脱敏执行日志。
- GR-12: 学生学习 App 账号密码只用于当次服务调用，不保存明文，不向任何角色展示密码。
- GR-13: 执行日志展示前必须脱敏，过滤密码、Token、Cookie、请求头、账号明文等敏感字段。
- GR-14: 服务提交记录保留 180 天。
- GR-15: 审计记录仅管理员可查看。
- GR-16: 大屏看板对所有人公开，但只展示聚合统计。
- GR-17: 统计记录文档仅管理员可导出。
- GR-18: 访客留言后立即公开，管理员可删除或隐藏不合适留言。

## 3. 操作定义

| 操作 | 含义 |
| --- | --- |
| 查看 | 查看页面、列表、详情、公开数据 |
| 注册 / 登录 | 使用 Lumitime 站点账号体系进入系统 |
| 新增 | 创建内容、邀请码、工作站服务、配置项 |
| 编辑 | 修改已有内容或配置 |
| 删除 | 删除内容、附件、留言、配置项 |
| 发布 / 下架 | 控制脚本、作品、博客、服务是否对目标用户可见 |
| 留言 | 在随记 / 留言板提交公开留言 |
| 使用服务 | 在工作站服务页发起一次服务请求 |
| 查看记录 | 查看服务提交记录或运行结果 |
| 复制 | 复制脚本代码 |
| 下载 | 下载被允许的作品压缩包或附件 |
| 导出 | 导出统计记录、服务记录或后台数据 |
| 授权 / 配置 | 配置邀请码、内容可见性、作品下载权限、工作站服务参数 |
| 查看审计 | 查看操作日志、服务请求追溯记录、安全记录 |

## 4. 资源权限矩阵

### R1. 登录 / 注册

| 角色 | 权限 |
| --- | --- |
| 访客 | 可访问登录页；可使用邀请码注册；可使用 Lumitime 站点账号登录 |
| 邀请用户 | 可登录、退出；可修改自己的站点密码和基础资料 |
| 管理员 | 可登录、退出；可管理站点账号体系；可重置或禁用用户账号 |

### R2. 主页

| 角色 | 权限 |
| --- | --- |
| 访客 | 可查看公开主页；可看到公开入口和登录 / 注册入口；邀请内容显示为需登录 |
| 邀请用户 | 可查看主页；可进入脚本、作品、博客、工作站、随记、大屏看板 |
| 管理员 | 可查看主页；可进入全部模块和管理后台 |

### R3. 邀请码

| 角色 | 权限 |
| --- | --- |
| 访客 | 可在注册时使用有效邀请码 |
| 邀请用户 | 不可查看、创建、禁用邀请码 |
| 管理员 | 可创建、查看、禁用、设置过期时间、设置使用次数、查看使用记录 |

### R4. 用户

| 角色 | 权限 |
| --- | --- |
| 访客 | 无用户管理权限 |
| 邀请用户 | 可查看和修改自己的基础资料；不可查看其他用户 |
| 管理员 | 可查看用户列表；可禁用用户；可查看用户注册时间、最后登录时间、角色和状态 |

### R5. 脚本分享

| 角色 | 权限 |
| --- | --- |
| 访客 | 不可查看 |
| 邀请用户 | 可查看脚本列表和详情；可复制代码 |
| 管理员 | 可新增、编辑、删除、发布、下架脚本；可查看脚本访问情况 |

实现约束：

- 脚本只提供代码文本，不提供文件下载。
- 不做“禁止复制”设计，避免制造虚假的安全感。

### R6. 个人作品

| 角色 | 权限 |
| --- | --- |
| 访客 | 不可查看 |
| 邀请用户 | 可查看全部已发布个人作品 |
| 管理员 | 可新增、编辑、删除、发布、下架作品；可上传、替换、删除作品压缩包 |

### R7. 个人作品压缩包 / 附件

| 角色 | 权限 |
| --- | --- |
| 访客 | 不可查看、不可下载 |
| 邀请用户 | 可下载管理员配置为允许下载的作品压缩包；不可下载未授权附件 |
| 管理员 | 可上传、替换、删除附件；可按作品配置是否允许邀请用户下载 |

### R8. 经验心得 / 技术博客

| 角色 | 权限 |
| --- | --- |
| 访客 | 不可查看博客正文 |
| 邀请用户 | 可查看全部已发布博客 |
| 管理员 | 可新增、编辑、删除、发布、下架博客 |

### R9. 随记 / 留言板

| 角色 | 权限 |
| --- | --- |
| 访客 | 可查看所有公开留言；可提交留言；留言提交后立即公开 |
| 邀请用户 | 可查看所有公开留言；可提交留言 |
| 管理员 | 可查看全部留言；可删除、隐藏、恢复留言 |

实现约束：

- 留言虽立即公开，但应保留基础防刷策略，例如频率限制、内容长度限制、敏感词或垃圾内容拦截。

### R10. 工作站入口

| 角色 | 权限 |
| --- | --- |
| 访客 | 不可查看 |
| 邀请用户 | 可查看工作站入口；可查看首版全部工作站服务 |
| 管理员 | 可查看工作站入口；可创建、编辑、删除、启用、停用工作站服务 |

实现约束：

- 首版所有邀请用户都能看到全部工作站服务。
- 后续预留“单服务授权”能力，但首版不要求实现。

### R11. 工作站服务

| 角色 | 权限 |
| --- | --- |
| 访客 | 不可查看、不可使用 |
| 邀请用户 | 可进入服务页；可按服务要求提交必要信息；可发起服务请求 |
| 管理员 | 可配置服务名称、说明、输入项、运行参数、启停状态、返回日志展示规则 |

### R12. 日志自动提交服务

| 角色 | 权限 |
| --- | --- |
| 访客 | 不可查看、不可使用 |
| 邀请用户 | 可进入服务页；可输入学生学习 App 账号和密码；可点击自动提交；可查看自己的脱敏执行日志 |
| 管理员 | 可配置服务；可接入后端自动化脚本；可查看全部提交记录和脱敏执行日志 |

实现约束：

- 学生学习 App 账号密码不同于 Lumitime 登录账号密码。
- 学生学习 App 密码不保存、不记录、不展示、不导出。
- 学生学习 App 账号不保存明文；如需追溯，只保存不可逆哈希和可选掩码。
- 后台运行日志返回给用户前必须脱敏。

### R13. 服务提交记录

| 角色 | 权限 |
| --- | --- |
| 访客 | 不可查看 |
| 邀请用户 | 可查看自己的提交记录、状态、提交时间、request_id、脱敏执行日志 |
| 管理员 | 可查看全部用户提交记录；可按用户、服务、时间、request_id 检索 |

实现约束：

- 提交记录保留 180 天。
- 用户只能看自己的提交记录。
- 管理员查看的记录也不得包含学生学习 App 密码。

### R14. 审计记录

| 角色 | 权限 |
| --- | --- |
| 访客 | 不可查看 |
| 邀请用户 | 不可查看 |
| 管理员 | 可查看系统审计记录、邀请码使用记录、服务请求追溯记录、敏感操作记录 |

实现约束：

- 审计记录中不得包含密码、Token、Cookie、完整请求头或学生学习 App 账号明文。
- 审计记录应包含 request_id、Lumitime 用户 ID、服务 ID、时间、来源 IP 哈希、User-Agent、结果状态等追溯字段。

### R15. 大屏看板

| 角色 | 权限 |
| --- | --- |
| 访客 | 可查看公开聚合统计 |
| 邀请用户 | 可查看公开聚合统计 |
| 管理员 | 可查看公开聚合统计；可查看后台统计配置 |

看板公开指标：

- 网站用户数
- 开发者数
- 访问数
- 个人作品数
- 脚本数
- 博客数
- 随记数
- 工作组 / 工作站服务数
- 上述指标的日度变化

实现约束：

- 大屏看板只展示聚合统计。
- 不展示用户列表、IP、账号、留言原文、提交日志明细等隐私数据。

### R16. 统计记录文档

| 角色 | 权限 |
| --- | --- |
| 访客 | 不可导出 |
| 邀请用户 | 不可导出 |
| 管理员 | 可导出统计记录文档 |

实现约束：

- 统计记录文档只包含聚合指标、快照日期、日度变化、生成时间。
- 文档不得包含个人敏感信息、原始访问日志、服务提交明细。

## 5. 服务类权限补充

工作站服务是 Lumitime 的关键扩展点。首版服务模型如下：

| 项目 | 规则 |
| --- | --- |
| 服务可见性 | 首版所有邀请用户可见 |
| 服务使用权 | 邀请用户可使用，访客不可使用 |
| 服务管理权 | 仅管理员 |
| 服务输入 | 按服务定义，例如学生学习 App 账号、密码 |
| 服务执行 | 后端自动运行管理员封装的脚本 |
| 执行结果 | 返回成功 / 失败状态和脱敏执行日志 |
| 用户记录 | 邀请用户只能查看自己的提交记录 |
| 管理员记录 | 管理员可查看全部提交记录 |
| 保留周期 | 180 天 |
| 后续扩展 | 支持按单个服务配置可访问用户 |

## 6. 日志自动提交服务权限流

```text
邀请用户登录 Lumitime
  ↓
进入工作站
  ↓
打开“日志自动提交”服务
  ↓
输入学生学习 App 账号和密码
  ↓
点击自动提交
  ↓
后端生成 request_id
  ↓
后端调用自动化脚本
  ↓
系统过滤敏感信息
  ↓
返回脱敏执行日志给用户
  ↓
保存非敏感提交记录，保留 180 天
```

权限和隐私要求：

- 邀请用户只能看到自己的本次执行结果和历史记录。
- 管理员可查看所有用户的提交状态和脱敏日志。
- 任何角色都不能查看学生学习 App 密码。
- 系统不能保存学生学习 App 密码。
- 系统不能把学生学习 App 账号密码写入日志、统计、导出文件或错误追踪系统。

## 7. 测试关注点

后续测试用例应覆盖以下权限边界：

- 访客无法访问脚本、作品、博客和工作站。
- 访客可以查看公开留言和大屏看板。
- 访客留言后立即公开。
- 邀请用户可以查看全部已发布脚本、作品、博客。
- 邀请用户可以复制脚本代码。
- 邀请用户不能新增、编辑、删除、发布、下架内容。
- 邀请用户只能下载被管理员配置为允许下载的作品压缩包。
- 邀请用户可以进入工作站并使用服务。
- 邀请用户只能查看自己的服务提交记录。
- 管理员可以查看全部服务提交记录。
- 执行日志对邀请用户和管理员展示前均已脱敏。
- 学生学习 App 密码不出现在数据库、日志、审计、导出文档、浏览器存储中。
- 统计记录文档只有管理员可导出。
- 审计记录只有管理员可查看。

## 8. 待确认项

- 开发者数的统计来源：是否由管理员手动标记开发者，还是固定为管理员数量。
- 作品压缩包下载是否需要记录下载次数和下载用户。
- 服务提交记录 180 天后是自动删除，还是转为归档脱敏统计。
- 工作站服务后续按用户授权时，是否需要用户组概念。
- 管理员是否需要手动重跑某个用户的服务请求；如果需要，必须重新获得用户授权和账号密码输入。
