# 风险清单与应对方案: 拾光筑梦 Lumitime

版本：v0.1  
日期：2026-06-12  
适用阶段：需求评审 / 技术评审 / 开发过程 / 上线评审 / 运维复盘  
关联文档：PRD、权限矩阵、概要设计、详细设计、测试计划、上线部署与运维手册

## 1. 文档目的

本文档用于提前识别 Lumitime 项目中的产品、技术、安全、隐私、运维和交付风险，并明确预防措施、监控信号和应急处理方案。

风险分级：

| 等级 | 定义 |
| --- | --- |
| 高 | 可能导致隐私泄露、权限越权、数据丢失、核心功能不可用或阻塞上线 |
| 中 | 可能影响用户体验、开发进度、运维成本或局部功能稳定性 |
| 低 | 影响较小，可通过后续优化解决 |

状态定义：

| 状态 | 含义 |
| --- | --- |
| Open | 风险存在，需持续关注 |
| Mitigated | 已有明确缓解措施 |
| Deferred | 后续版本处理 |
| Closed | 风险已消除 |

## 3. 详细风险与应对

### RISK-001: 学生学习 App 密码被保存或写入日志

等级：高  
类别：隐私安全  
状态：Mitigated

风险描述：

日志自动提交服务需要用户输入学生学习 App 账号密码。如果密码被写入数据库、应用日志、执行日志、审计记录或导出文件，将造成严重隐私风险。

可能影响：

- 用户隐私泄露。
- 系统信任受损。
- 后续问题追责困难。

预防措施：

- 数据模型中不设置学生密码字段。
- 创建服务请求时只在请求生命周期中使用密码。
- 脚本输出先脱敏再存储。
- 导出使用字段白名单。
- 测试用例覆盖数据库、接口响应、日志、导出文件。

监控信号：

- 日志中出现 `password`、`pwd`、`student_password` 等敏感关键词。
- 导出文件中出现疑似密码字段。

应急处理：

1. 立即停用日志自动提交服务。
2. 下线相关日志和导出文件访问。
3. 排查泄露范围。
4. 清理敏感记录。
5. 修复脱敏逻辑并重新执行隐私安全测试。

### RISK-002: 邀请用户查看他人服务提交记录

等级：高  
类别：权限安全  
状态：Mitigated

风险描述：

邀请用户通过修改 URL 或接口参数访问其他用户的服务提交记录。

预防措施：

- 后端查询绑定 `lumitime_user_id = current_user.id`。
- 管理员和邀请用户使用不同查询逻辑。
- 测试用例 RECORD-002 覆盖越权访问。

应急处理：

- 立即修复查询条件。
- 检查审计记录确认是否发生越权访问。
- 必要时通知受影响用户。

### RISK-003: 访客访问邀请内容或后台

等级：高  
类别：权限安全  
状态：Mitigated

风险描述：

访客绕过前端页面，直接访问脚本、作品、博客、工作站或后台接口。

预防措施：

- 所有受限 API 使用统一鉴权中间件。
- 管理员接口强制 `role=admin`。
- 前端只做体验控制，后端才是权限边界。

应急处理：

- 禁用受影响接口。
- 修复鉴权中间件或路由配置。
- 执行权限回归测试。

### RISK-004: 原始脚本日志包含敏感信息

等级：高  
类别：隐私安全  
状态：Mitigated

风险描述：

自动化脚本可能输出账号、密码、Cookie、Token、请求头或异常堆栈。

预防措施：

- Worker 先脱敏再保存。
- 不保存原始未脱敏日志。
- 脱敏采用字段白名单 + 关键字正则双重策略。

应急处理：

- 停止日志展示和导出。
- 清理已保存敏感日志。
- 增强脱敏规则。
- 回归 PRIV-003、PRIV-004、EXPORT-003。

### RISK-005: 作品附件被未授权下载

等级：高  
类别：内容安全  
状态：Mitigated

风险描述：

个人作品压缩包可能包含项目、软著、专利、论文等资料，如果下载地址公开或权限判断缺失，会导致未授权下载。

预防措施：

- 下载必须走后端接口鉴权。
- 不暴露永久公开地址。
- 邀请用户下载受 `allow_download` 控制。

应急处理：

- 立即禁用附件下载接口。
- 更换暴露的文件链接。
- 检查访问日志和下载记录。

### RISK-006: 真实日志自动提交脚本接入后失败率高

等级：高  
类别：工作站服务  
状态：Open

风险描述：

真实学校系统可能存在验证码、风控、登录限制、网络不稳定或页面变更，导致脚本失败。

预防措施：

- 首版先使用模拟脚本验证服务框架。
- 真实接入放入 V0.3。
- 脚本错误映射到标准失败原因分类。
- 增加服务启停开关。

应急处理：

- 停用真实服务。
- 回退到“服务暂不可用”提示。
- 保留历史记录。
- 修复脚本后灰度启用。

### RISK-007: 学校系统改版导致脚本不可用

等级：高  
类别：外部依赖  
状态：Open

预防措施：

- 脚本版本化。
- 保存脚本版本到服务记录。
- 失败率异常时告警。
- 保留服务开关。

应急处理：

- 停用服务。
- 更新脚本适配。
- 补充测试用例。

### RISK-008: Worker 与 Web 分离后临时凭证传递不安全

等级：高  
类别：架构安全  
状态：Open

风险描述：

学生学习 App 密码不能落库，但 Worker 独立执行脚本，需要安全传递当次凭证。

预防措施：

- 详细设计中将临时凭证传递列为待技术确认项。
- 禁止把密码写入数据库任务表。
- 首版实现前进行专项技术评审。

可选方案：

- Web 与 Worker 使用内存安全队列。
- 使用短期加密缓存并设置极短 TTL。
- Worker 由 Web 进程安全派发任务。

应急处理：

- 若发现凭证持久化，立即停用服务。
- 清理凭证数据。
- 重构传递机制。

### RISK-009: Docker 更新导致数据或附件丢失

等级：高  
类别：运维  
状态：Mitigated

预防措施：

- 数据库和附件使用持久化卷或外部存储。
- 容器只放代码和运行环境。
- 更新前手动备份。
- 镜像使用版本号。

应急处理：

- 停止服务。
- 使用更新前备份恢复。
- 回滚到上一稳定镜像。

### RISK-010: 数据库迁移失败导致服务不可用

等级：高  
类别：运维 / 数据  
状态：Open

预防措施：

- 迁移前备份。
- 迁移脚本评审。
- 高风险迁移准备回滚脚本。
- 预发布环境先执行。

应急处理：

- 停止新版本。
- 回滚镜像。
- 恢复数据库备份或执行回滚迁移。

### RISK-011: 留言立即公开导致垃圾内容或不当内容

等级：中  
类别：内容治理  
状态：Open

预防措施：

- 频率限制。
- 内容长度限制。
- 管理员可隐藏 / 删除。

后续增强：

- 敏感词拦截。
- 图片或链接限制。
- 默认待审核开关。

### RISK-012: 大屏访问数统计口径不清导致数据争议

等级：中  
类别：数据统计  
状态：Open

预防措施：

- 在后续统计口径文档中明确 PV、UV、登录用户访问数。
- 大屏展示指标说明。

### RISK-013: 登录页动效影响性能或可用性

等级：中  
类别：视觉交互  
状态：Mitigated

预防措施：

- 动效总时长控制在 1.2s 到 2.0s。
- 支持减少动态效果降级。
- 移动端简化。
- 核心登录流程不依赖动画。

### RISK-014: 小猫交互影响登录输入

等级：低  
类别：视觉交互  
状态：Deferred

预防措施：

- 小猫交互放到二期。
- 用户输入时自动停止跟随。
- 不遮挡表单。

### RISK-015: CSV 导出包含敏感字段

等级：高  
类别：隐私安全  
状态：Mitigated

预防措施：

- 导出字段白名单。
- 禁止导出密码、完整账号、Token、Cookie、Authorization。
- 测试用例 EXPORT-003 覆盖。

### RISK-016: 备份失败未被发现

等级：高  
类别：运维  
状态：Open

预防措施：

- 备份任务写日志。
- 后台或运维检查最近一次备份状态。
- 后续增加告警渠道。

应急处理：

- 立即手动备份。
- 排查磁盘、权限、脚本错误。
- 修复后执行恢复演练。

### RISK-017: 服务器磁盘被附件、导出或备份占满

等级：中  
类别：运维  
状态：Open

预防措施：

- 限制上传文件大小。
- 定期清理过期导出文件。
- 备份设置保留周期。
- 监控磁盘空间。

### RISK-018: 首版范围膨胀导致进度失控

等级：中  
类别：项目管理  
状态：Mitigated

预防措施：

- 开发计划明确 V0.1 / V0.2 / V0.3。
- 小猫交互后置。
- 真实脚本接入后置。
- 服务级授权后置。

### RISK-019: 开发者数统计口径未定

等级：低  
类别：数据统计  
状态：Open

建议方案：

- 首版可固定为管理员数量。
- 或由管理员手动标记开发者。
- 最终口径需在看板统计口径中确认。

### RISK-020: 未开启 HTTPS 导致凭证传输风险

等级：高  
类别：安全  
状态：Mitigated

预防措施：

- 运维手册要求必须开启 HTTPS。
- HTTP 自动跳转 HTTPS。
- 上线检查清单包含 HTTPS 验证。

## 4. 上线前高风险检查清单

上线前必须确认：

- 学生学习 App 密码不入库。
- 学生学习 App 密码不进入应用日志。
- 学生学习 App 密码不进入执行日志。
- 学生学习 App 密码不进入 CSV 导出。
- 学生完整账号不入库，仅保存掩码和不可逆哈希。
- 邀请用户不能查看他人提交记录。
- 访客不能访问邀请内容和后台。
- 管理员接口全部强制鉴权。
- HTTPS 已启用。
- 数据库和附件已持久化。
- 每日备份已配置。
- 更新前备份流程已验证。
- 回滚流程已验证。
