本文档定义 Lumitime 首版系统的技术架构、技术栈、模块边界、部署方式、工作站服务执行机制和安全约束，作为开发实施和技术评审的统一依据。

## 2. 系统定位

Lumitime 是一个个人工作站网页，用于沉淀个人脚本、作品、技术经验、随记留言和工作站服务。

系统首版包含三类访问角色：

| 角色 | 权限定位 |
| --- | --- |
| 访客 | 访问主页、随记 / 留言板、公开大屏看板，提交公开留言 |
| 邀请用户 | 访问脚本、作品、博客、工作站服务和自己的提交记录 |
| 管理员 | 管理内容、邀请码、用户、留言、工作站服务、提交记录、审计和导出 |

首版核心目标：

- 完成个人工作站的公开区、邀请用户区和管理后台。
- 跑通日志自动提交服务的模拟闭环。
- 建立权限、审计、脱敏、导出和部署基础。
- 为后续真实脚本接入和服务拆分预留架构空间。

## 3. 架构原则

| 原则 | 说明 |
| --- | --- |
| 模块化单体起步 | 首版采用模块化单体，降低开发和部署复杂度 |
| 前后端分离 | 前端和后端独立工程、独立构建、通过 API 协作 |
| 后端强制权限 | 前端只控制显示体验，所有受限数据和操作均由后端鉴权 |
| Cookie 会话 | 使用服务端 Cookie Session 管理登录态 |
| 数据最小化 | 学生学习 App 密码不入库、不写日志、不导出、不进入审计 |
| 日志先脱敏后存储 | 执行日志、异常信息和导出内容必须经过统一脱敏 |
| 异步任务闭环 | 工作站服务通过请求记录、Worker、状态轮询和执行日志形成闭环 |
| 单机容器化部署 | 首版使用单台云服务器和 Docker Compose 部署 |
| 可演进架构 | 后续可按业务边界拆分为微服务或独立任务执行服务 |

## 4. 技术栈

### 4.1 前端

| 项目 | 选型 |
| --- | --- |
| 框架 | React |
| 构建工具 | Vite |
| 语言 | TypeScript |
| 样式方案 | Tailwind CSS 或 CSS Modules，按最终 UI 工程规范落地 |
| 路由 | React Router |
| 数据请求 | Fetch / Axios 封装统一 API Client |
| 状态管理 | 优先使用 React Query 管理服务端状态；局部 UI 状态使用 React 内置状态 |
| 生产托管 | React 构建产物由 Nginx 托管 |

前端职责：

- 实现公开区、邀请用户区和管理后台页面。
- 根据当前用户角色控制导航和入口展示。
- 调用 `/api/v1` 后端接口。
- 提供日志自动提交服务表单、状态卡片、轮询和提交记录视图。
- 不在浏览器持久化学生学习 App 账号密码。

### 4.2 后端

| 项目 | 选型 |
| --- | --- |
| 框架 | FastAPI |
| 语言 | Python |
| API 风格 | RESTful API |
| API 前缀 | `/api/v1` |
| 数据校验 | Pydantic |
| 后台任务 | 独立 Worker 进程 |
| 脚本适配 | Python 模块适配层 |

后端职责：

- 认证、注册、会话和权限控制。
- 内容、邀请码、用户、留言、工作站服务和审计管理。
- 工作站服务请求创建、状态查询、失败重试和记录导出。
- 大屏聚合指标和日度快照。
- 日志脱敏、敏感字段过滤和导出白名单。

### 4.3 数据库

| 项目 | 选型 |
| --- | --- |
| 数据库 | PostgreSQL |
| 数据访问 | SQLAlchemy 2.x |
| 迁移工具 | Alembic |

PostgreSQL 用于存储：

- 用户、邀请码和会话相关数据。
- 内容、附件元数据和留言。
- 工作站服务配置、服务请求、执行日志和审计记录。
- 访问事件、日度统计快照和导出任务。

### 4.4 会话与认证

| 项目 | 选型 |
| --- | --- |
| 登录态 | Cookie Session |
| Cookie 属性 | `HttpOnly`、`Secure`、`SameSite=Lax` 或按部署域名策略配置 |
| 密码存储 | 仅保存哈希 |
| 密码哈希 | Argon2id，运行环境不满足时使用 bcrypt |
| CSRF | 对变更类请求启用 CSRF 防护或 SameSite + CSRF Token 组合策略 |

会话规则：

- Lumitime 站点账号与学生学习 App 账号完全分离。
- 管理员、邀请用户通过同一登录接口进入系统。
- 管理员登录后进入管理后台。
- 邀请用户登录后进入主页。
- 访客不进入 `users` 表。

## 6. 后端模块设计

### 6.1 认证与会话模块

职责：

- Lumitime 站点账号登录。
- 邀请码注册。
- 退出登录。
- 当前用户查询。
- 修改站点密码。

安全要求：

- 登录失败统一提示，避免账号枚举。
- 密码只保存哈希。
- Session Cookie 设置安全属性。
- 变更类接口具备 CSRF 防护。

### 6.2 权限控制模块

权限校验分四层：

```text
前端导航和路由显示控制
  ↓
API 鉴权中间件
  ↓
模块内角色判断
  ↓
数据查询范围约束
```

关键规则：

- 访客不能访问脚本、作品、博客、工作站和后台接口。
- 邀请用户不能访问任何管理员接口。
- 邀请用户只能查看自己的服务提交记录。
- 管理员可查看全部服务提交记录和完整脱敏执行日志。

### 6.3 内容模块

脚本、作品和博客使用统一内容模型，通过 `type` 区分：

- `script`
- `work`
- `blog`

规则：

- 管理员是唯一内容维护角色。
- 脚本只提供代码文本和复制能力，不提供脚本文件下载。
- 作品附件下载由 `allow_download` 控制。
- 邀请用户只能访问已发布内容。

### 6.4 留言模块

职责：

- 公开留言列表。
- 访客和登录用户提交留言。
- 管理员隐藏、恢复、删除留言。

规则：

- 留言默认立即公开。
- 提交接口必须具备频率限制、长度限制和基础内容校验。
- 来源 IP 只保存哈希。

### 6.5 工作站服务模块

首版工作站采用“服务集合”模型，不实现复杂工作区和表单中心。

首版服务：

- 日志自动提交。

规则：

- 所有邀请用户可查看启用的工作站服务。
- 管理员可创建、编辑、启用、停用服务。
- 服务提交后立即返回 `service_request_id`。
- 前端通过轮询查询执行状态。
- 失败记录允许重试，重试必须重新输入学生学习 App 账号密码。

### 6.6 看板统计模块

职责：

- 采集脱敏访问事件。
- 生成日度统计快照。
- 提供公开聚合指标。
- 支持管理员导出统计记录。

公开指标：

- 网站用户数。
- 开发者数。
- 访问数。
- 个人作品数。
- 脚本数。
- 博客数。
- 随记数。
- 工作站服务数。

规则：

- 大屏接口公开。
- 只返回聚合数据。
- 不返回用户列表、IP、账号、留言原文或服务日志。

## 7. 日志自动提交 MVP 闭环

### 7.1 范围

V0.1 使用模拟脚本跑通工作站服务闭环，不接入真实学校系统。

闭环包含：

- 创建服务请求。
- 状态流转。
- Worker 异步执行。
- 步骤日志生成。
- 日志脱敏。
- 用户结果摘要。
- 管理员完整脱敏日志。
- 失败重试。
- 服务记录导出。

V0.1 不执行：

- 真实 SSO 登录。
- 真实验证码 OCR。
- 真实教务 doSave。
- 真实学校系统联调。

### 7.2 状态流转

```text
pending
  ↓
running
  ↓
success / failed / timeout / not_integrated
```

失败分类：

| 错误码 | 含义 |
| --- | --- |
| AUTH_FAILED | 学生学习 App 账号或密码错误 |
| NETWORK_ERROR | 网络异常 |
| SCHOOL_SYSTEM_ERROR | 学校系统异常 |
| SCRIPT_ERROR | 脚本异常 |
| VALIDATION_ERROR | 输入校验失败 |
| TIMEOUT | 执行超时 |
| UNKNOWN_ERROR | 未知错误 |

### 7.3 请求创建流程

```text
邀请用户提交服务表单
  ↓
后端校验登录态和服务可用状态
  ↓
校验输入 schema
  ↓
生成 service_request_id
  ↓
保存服务请求记录
  ↓
保存学生账号掩码和不可逆哈希
  ↓
丢弃学生学习 App 密码
  ↓
写入审计记录
  ↓
返回 service_request_id
  ↓
Worker 拉取任务并执行模拟脚本
```

V0.1 模拟执行阶段不依赖真实学生学习 App 密码。密码仅用于满足产品表单和安全链路测试，不进入数据库、任务表、执行日志、审计或导出。

### 7.4 前端轮询

推荐轮询规则：

- 轮询间隔：2 秒。
- 最大前台等待：120 秒。
- 终态为 `success`、`failed`、`timeout`、`not_integrated` 时停止轮询。
- 超过前台等待时间后提示用户稍后到提交记录查看。

## 8. `submit_example` 集成方案

### 8.1 文件作用

| 文件 | 作用 |
| --- | --- |
| `main.py` | 本地 MVP FastAPI 入口，包含任务创建、内存任务状态和 SSE |
| `orchestrator_stub.py` | 执行编排，支持干跑和真实 SSO 链路 |
| `sso_cppu.py` | 中国人民警察大学 SSO 登录辅助 |
| `jw_chain.py` | 教务 H5 链路与 doSave 提交 |
| `deepseek_sxrz.py` | 调用 DeepSeek API 生成实习日志正文 |
| `log_library.py` | 从本地日志库抽取实习日志正文 |
| `han_text.py` | 汉字计数与截断 |
| `mock_repo.py` | 简单案例库读取 |

### 8.2 集成方式

`submit_example/main.py` 不作为 Lumitime 生产后端入口。Lumitime 后端只复用其中的执行模型和脚本逻辑，落地为工作站服务适配器。

集成规则：

- 使用 Lumitime 的 `service_request_id` 替代本地 MVP 的 `task_id`。
- 使用 PostgreSQL 服务请求表替代本地内存 `_tasks`。
- 使用独立 Worker 替代本地 `asyncio.create_task`。
- 使用统一日志脱敏模块替代脚本内局部日志处理。
- 使用服务请求状态接口替代 SSE 作为首版主状态查询机制。
- DeepSeek 仅作为日志正文生成的可选 API 调用点处理，不作为独立架构模块。

### 8.3 输入映射

| Lumitime 字段 | 脚本字段 | 处理规则 |
| --- | --- | --- |
| `student_account` | `jw_username` / `student_no` | 当次请求校验后生成掩码和不可逆哈希 |
| `student_password` | `jw_password` | 当次请求使用，不持久化 |
| `display_name` | `display_name` | 作为服务输入项，由用户填写 |
| `task_config.log_dates` | `log_dates` | 用于模拟多日提交 |
| `task_config.sxrz_text` | `sxrz_text` | 手写实习日志正文 |
| `task_config.station_activity_text` | `station_activity_text` | 用于生成日志正文 |
| `task_config.deepseek_api_key` | `deepseek_api_key` | 可选 API Key，仅当次请求使用 |
| `task_config.pacing_total_sec` | `pacing_total_sec` | 模拟多日提交节奏 |
| `task_config.request_spacing_sec` | `request_spacing_sec` | 模拟请求间隔 |

### 8.4 模拟结果

V0.1 Worker 需要支持通过测试配置触发以下结果：

- success。
- AUTH_FAILED。
- NETWORK_ERROR。
- SCHOOL_SYSTEM_ERROR。
- SCRIPT_ERROR。
- TIMEOUT。
- not_integrated。

测试触发方式通过服务配置或测试专用字段实现，生产界面不暴露明显的调试选项。

## 9. 敏感信息处理

### 9.1 学生学习 App 凭证

| 信息 | 处理方式 |
| --- | --- |
| 学生学习 App 账号 | 不保存明文；保存掩码和不可逆哈希 |
| 学生学习 App 密码 | 不保存、不哈希、不掩码、不导出 |
| Cookie / Token / Authorization | 不保存；日志中必须脱敏 |
| 原始请求体 | 不保存 |
| 原始执行日志 | 不保存 |

### 9.2 日志脱敏

日志进入数据库前必须执行脱敏。

必须脱敏：

- `password`
- `pwd`
- `student_password`
- 完整学生账号
- `token`
- `cookie`
- `authorization`
- `bearer`
- 原始请求头
- 可能包含敏感字段的异常堆栈

### 9.3 导出白名单

服务记录导出允许字段：

- `service_request_id`
- 服务名称
- Lumitime 用户 ID 或展示名
- 状态
- 失败原因分类
- 结果摘要
- 学生账号掩码
- 提交时间
- 开始时间
- 结束时间
- 耗时
- 脚本版本
- 完整脱敏日志

禁止导出：

- 学生学习 App 密码
- 学生完整账号
- Cookie
- Token
- Authorization
- 原始请求头
- 原始未脱敏日志

核心数据表：

| 表 | 用途 |
| --- | --- |
| `users` | Lumitime 站点用户 |
| `invite_codes` | 邀请码 |
| `invite_code_usages` | 邀请码使用记录 |
| `content_items` | 脚本、作品、博客统一内容 |
| `content_attachments` | 作品附件 |
| `messages` | 随记 / 留言 |
| `workstation_services` | 工作站服务配置 |
| `service_requests` | 服务提交记录 |
| `service_execution_logs` | 脱敏执行日志 |
| `service_audits` | 服务审计 |
| `audit_logs` | 系统审计 |
| `visit_events` | 短期脱敏访问事件 |
| `daily_metric_snapshots` | 日度统计快照 |

保留策略：

- 内容长期保留，删除采用软删除。
- 服务请求、执行日志和服务审计保留 180 天。
- 日度统计快照长期保留。
- 访问明细短期保留，聚合后清理。
- 导出文件设置过期清理。

## 11. 文件存储

首版文件存储采用服务器本地持久化目录，通过文件存储适配层访问。

规则：

- 上传附件不得保存在容器内部临时目录。
- 附件下载必须经过后端鉴权。
- 不向前端暴露永久公开文件地址。
- 作品附件是否允许邀请用户下载由 `allow_download` 控制。
- 后续可将文件存储适配层切换为对象存储。

## 12. 部署架构

首版部署在单台云服务器，使用 Docker Compose 编排。服务器支持 HTTPS，生产流量统一先进入 Nginx，由 Nginx 托管前端静态资源并反向代理后端接口。

服务组成：

```text
nginx
api
worker
db
backup
```

职责：

| 服务 | 职责 |
| --- | --- |
| `nginx` | HTTPS、域名、反向代理、请求体大小限制、React 静态资源托管 |
| `api` | FastAPI 后端接口 |
| `worker` | 工作站服务任务执行 |
| `db` | PostgreSQL |
| `backup` | 数据库、附件和配置模板备份 |

持久化目录：

```text
/opt/lumitime
├─ data/
│  ├─ db/
│  ├─ uploads/
│  └─ exports/
├─ backups/
│  ├─ db/
│  ├─ uploads/
│  └─ configs/
└─ logs/
```

部署要求：

- 必须启用 HTTPS。
- HTTP 自动跳转 HTTPS。
- Nginx 负责 TLS 终止、前端静态资源托管和 `/api/v1` 反向代理。
- React 前端通过 Vite 构建为静态文件后发布到 Nginx 托管目录。
- `.env` 不提交代码仓库。
- 数据库和附件必须持久化。
- 更新前必须备份数据库和附件。
- 镜像使用版本号，不使用裸 `latest` 作为生产版本依据。

## 13. 架构演进

首版采用模块化单体，后续可根据访问量、任务并发和维护复杂度逐步拆分。

可拆分方向：

| 演进方向 | 触发条件 |
| --- | --- |
| Worker 独立扩容 | 工作站服务执行耗时增加或任务并发上升 |
| 消息队列 | 数据库任务表无法满足并发和可靠性要求 |
| 文件对象存储 | 附件增长明显或需要 CDN / 外部存储 |
| 独立认证服务 | 用户规模扩大或需要多系统登录 |
| 工作站服务微服务化 | 真实脚本数量增加、运行环境差异明显 |
| 异步导出服务 | 导出数据量变大或同步导出超时 |

微服务化前提：

- API 契约稳定。
- 数据边界清晰。
- 服务间鉴权策略明确。
- 日志脱敏和审计规范保持一致。

## 14. 开发实施顺序

推荐实施顺序：

1. 初始化前端 React 和后端 FastAPI 工程。
2. 配置 PostgreSQL、SQLAlchemy、Alembic 和基础迁移。
3. 实现认证、Cookie Session、邀请码注册和角色权限。
4. 实现公开主页、登录页、注册页、留言板和大屏基础页面。
5. 实现内容模型、前台内容展示和管理后台 CRUD。
6. 实现工作站服务配置、服务请求和提交记录。
7. 实现 Worker、模拟脚本适配器、状态流转和日志脱敏。
8. 实现服务记录、审计、CSV 导出和隐私字段检查。
9. 实现 Docker Compose 开发 / 测试部署。
10. 执行全量权限、安全、隐私、UAT 和上线前回归。

## 15. 上线门槛

上线前必须满足：

- 管理员、邀请用户、访客权限边界正确。
- 登录、邀请码注册、退出登录正常。
- 管理员可维护脚本、作品、博客、留言和工作站服务。
- 邀请用户可访问已授权内容和工作站服务。
- 日志自动提交模拟闭环可运行。
- 服务请求状态、结果摘要、失败重试和提交记录正常。
- 学生学习 App 密码不入库、不写日志、不导出、不在响应中出现。
- 学生学习 App 完整账号不入库，只保存掩码和不可逆哈希。
- 大屏只展示聚合指标。
- 管理员导出 CSV 不包含敏感字段。
- Docker Compose 测试环境部署通过。
- 数据库和附件备份流程验证通过。
- P0 测试用例全部通过。
