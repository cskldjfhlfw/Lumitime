# 视觉交互设计说明: 拾光筑梦 Lumitime

版本：v0.1  
日期：2026-06-12  
适用阶段：UI 设计 / 前端开发 / 动效设计 / 验收测试  
关联文档：PRD、权限矩阵、信息架构、接口设计、数据模型设计

## 1. 设计定位

Lumitime 是个人工作站，不是企业门户或内容社区。视觉应服务于两个目标：

- 个人表达：体现“拾光筑梦”的微光、时间、向上、转向、积累感。
- 工具效率：脚本、作品、博客、工作站、后台管理必须清晰、轻量、可用。

首版主视觉采用黑白主色调，以光束、留白、书写感、微弱动效作为记忆点。整体不追求复杂装饰，而是用少量关键画面建立情绪。

## 2. 视觉原则

| 原则 | 说明 |
| --- | --- |
| 黑白主调 | 页面以黑、白、灰为核心，不使用大面积彩色渐变 |
| 微光点缀 | 使用低强度光束、边缘光、局部高亮表达“lumi 微光” |
| 留白克制 | 普通页面保持简洁，避免信息拥挤 |
| 情绪入口 | 登录页承担品牌情绪和故事表达 |
| 工具清晰 | 工作站、后台、大屏以效率和可读性优先 |
| 动效有意义 | 动效必须服务“下坠到向上”的寓意，不做纯装饰动画 |

## 3. 色彩体系

### 3.1 主色

| 色彩 | 用途 |
| --- | --- |
| 纯黑 / 近黑 | 登录页背景、大屏背景、强对比区域 |
| 纯白 / 近白 | 文字、人物剪影、表单、普通页面背景 |
| 中性灰 | 辅助文字、边框、分隔线、禁用状态 |
| 微光灰白 | 光束、hover 高亮、聚焦状态 |

### 3.2 使用规则

- 普通前台页面以浅色为主，黑白灰构成。
- 登录页和大屏看板可以使用深色背景。
- 不使用高饱和大面积颜色。
- 成功、失败、警告状态可以使用低饱和辅助色，但面积要小。
- 关键按钮优先使用黑底白字或白底黑字。

## 4. 字体与标题风格

### 4.1 品牌标题

登录页标题“拾光筑梦 Lumitime”应具有毛笔书写的艺术效果。

要求：

- 中文标题有书写感、笔锋感。
- 英文 Lumitime 与中文标题协调，不抢主体。
- 标题应清晰可读，不为了艺术效果牺牲识别。
- 字体加载失败时回退到系统 serif / sans-serif 组合。

### 4.2 副标题

副标题可使用：

```text
Every faint light of time, paves the way for your dream.
```

要求：

- 字重轻。
- 字距正常，不做过度拉伸。
- 与标题形成主次关系。

### 4.3 正文和后台

- 普通正文、导航、后台管理使用清晰易读的系统字体。
- 后台不使用毛笔字体。
- 毛笔风格只用于品牌首屏、登录页或少量品牌展示区域。

## 5. 登录页设计

### 5.1 页面结构

登录页采用左右布局，空间比例为 **左 1 : 右 2**。

```text
┌──────────────────────────────────────────────┐
│ 左侧视觉区 1/3        │ 右侧登录与品牌区 2/3 │
│                       │                      │
│ 下坠人物 / 光束       │ 标题                 │
│ 边缘交互小猫          │ 副标题               │
│                       │ 登录表单             │
│                       │ 邀请码注册入口       │
└──────────────────────────────────────────────┘
```

### 5.2 左侧视觉区

左侧视觉区基于用户提供的参考画面：黑色空间、筒形光束、人物位于光束中。

首屏状态：

- 背景为深黑或近黑。
- 画面中有一束向下延伸的光。
- 白色人物剪影位于画面左侧或左侧视觉区中，呈“向下爬 / 下坠”状态。
- 人物不应干扰右侧登录表单。

寓意：

- 登录前是“误以为自己在向下”。
- 进入主页后通过反转动画表达“方向被重新理解”。
- “你以为的深渊，其实有一束光在指引你往上”。

### 5.3 右侧登录与品牌区

右侧区域包含：

- 品牌标题：拾光筑梦 Lumitime。
- 品牌副标题。
- Lumitime 站点账号输入框。
- Lumitime 站点密码输入框。
- 登录按钮。
- 邀请码注册入口。
- 简短错误提示区。

交互规则：

- 输入框聚焦时使用黑白高对比边框或微光描边。
- 登录按钮在输入不完整时禁用。
- 登录失败时显示统一错误提示，不暴露账号是否存在。
- 登录成功后触发进入主页的转场动画。

### 5.4 移动端适配

移动端不强制维持 1:2 横向结构。

建议：

- 顶部为压缩后的品牌视觉。
- 中部为标题和登录表单。
- 人物和光束动效降级为轻量静态或短动画。
- 小猫交互在移动端改为点击后短距离移动，不跟随鼠标。

## 6. 登录到主页的反转动画

### 6.1 动画目标

登录成功后，画面从“向下”转为“向上”，完成视觉叙事转折。

核心表达：

```text
下坠 / 向下爬
  ↓
画面反转
  ↓
人物朝向上方
  ↓
光束成为向上的引导
  ↓
进入主页
```

### 6.2 动画流程

建议流程：

1. 用户点击登录。
2. 登录按钮进入 loading 状态。
3. 登录成功后左侧视觉区短暂聚焦。
4. 光束亮度轻微增强。
5. 人物方向从向下转为向上。
6. 画面完成一次平滑反转或视角转换。
7. 主页内容淡入。

### 6.3 动画时长

建议：

- 总时长控制在 1.2s 到 2.0s。
- 不超过 2.5s，避免影响登录效率。
- 用户多次登录或频繁跳转时可跳过动画。

### 6.4 降级规则

以下场景应降级：

- 用户系统开启减少动态效果。
- 移动端性能较低。
- 网络或资源加载失败。
- 浏览器不支持所需动画能力。

降级后：

- 显示静态光束和人物。
- 登录成功后直接淡入主页。

## 7. 小猫交互设计

### 7.1 角色定位

页面边缘放置一只 Q 版小猫，作为轻量互动元素。它不承担核心功能，不应影响登录、注册、工作站操作。

### 7.2 默认状态

- 小猫位于画面边缘。
- 默认保持静止或轻微呼吸动效。
- 不遮挡登录表单、按钮、错误提示。

### 7.3 点击后行为

用户点击小猫后：

- 小猫进入“跟随鼠标”状态。
- 小猫朝鼠标方向移动，而不是瞬移。
- 移动应有缓动，体现“跟着跑”的感觉。
- 小猫始终保持在视口内。

### 7.4 退出跟随

建议提供以下退出方式：

- 再次点击小猫，停止跟随。
- 鼠标离开页面一段时间后停止跟随。
- 用户开始输入账号密码时，小猫自动停止并回到边缘。

### 7.5 约束

- 不影响表单输入。
- 不触发表单失焦导致用户误操作。
- 不遮挡按钮。
- 不记录用户鼠标轨迹。
- 移动端改为点击后短动画，不持续跟随手指。

## 8. 主页设计

### 8.1 主页定位

主页对所有人公开，但邀请内容对访客显示锁定状态。

页面结构：

- 顶部导航。
- 品牌介绍区。
- 模块入口区。
- 公开入口：随记、大屏看板。
- 邀请入口：脚本分享、个人作品、经验心得、工作站。

### 8.2 访客状态

访客可见：

- 主页内容。
- 随记入口。
- 大屏看板入口。
- 登录 / 邀请码注册。
- 脚本、作品、博客、工作站的锁定入口。

访客点击锁定入口：

- 展示登录提示。
- 提供登录和邀请码注册按钮。

### 8.3 邀请用户状态

邀请用户可见并可进入：

- 脚本分享。
- 个人作品。
- 经验心得。
- 工作站。
- 随记。
- 大屏看板。
- 个人中心。

### 8.4 管理员状态

管理员额外可见：

- 管理后台入口。

登录成功后默认进入后台总览，但可以返回主页。

## 9. 工作站首页设计

### 9.1 展示方式

工作站首页使用服务卡片网格。

每个服务卡片包含：

- 服务名称。
- 服务简介。
- 服务状态。
- 最近更新时间。
- 进入按钮。

首版所有邀请用户可见全部启用服务。

### 9.2 我的提交记录入口

工作站首页提供“我的提交记录”入口。

展示建议：

- 最近 3 到 5 条提交记录摘要。
- 查看全部记录按钮。
- 状态标识：执行中、成功、失败、超时。

## 10. 日志自动提交服务页设计

### 10.1 页面布局

采用纵向布局：

```text
服务说明
  ↓
账号密码输入区
  ↓
自动提交按钮
  ↓
执行中 / 结果状态卡片
  ↓
最近提交记录
```

### 10.2 输入区

字段：

- 学生学习 App 账号。
- 学生学习 App 密码。

提示：

- 该账号密码不同于 Lumitime 登录账号密码。
- 系统仅用于当次提交，不保存密码。

### 10.3 提交反馈

提交后进入“执行中”状态卡片。

状态卡片展示：

- 当前状态。
- request_id / service_request_id。
- 等待提示。
- 轮询状态。

完成后展示：

- 成功 / 失败。
- 结果摘要。
- 失败原因分类。
- 重新提交按钮。

### 10.4 失败重试

失败记录允许重新提交，但必须重新输入学生学习 App 账号和密码。

不允许：

- 自动复用历史密码。
- 从历史记录恢复密码。
- 在页面上展示完整账号或密码。

## 11. 提交记录页面设计

### 11.1 用户记录列表

邀请用户可查看自己的提交记录。

列表字段：

- 服务名称。
- 提交时间。
- 状态。
- service_request_id。
- 耗时。
- 结果摘要。
- 账号掩码。

### 11.2 用户记录详情

展示：

- 状态。
- 失败原因分类。
- 结果摘要。
- 账号掩码。
- 是否可重试。

不展示：

- 完整执行日志。
- 学生学习 App 完整账号。
- 学生学习 App 密码。

## 12. 大屏看板设计

### 12.1 风格

大屏看板采用深色全屏风格。

视觉特征：

- 黑色或近黑背景。
- 白色数字指标。
- 灰白趋势线。
- 少量微光效果。
- 数据模块之间保持清晰间距。

### 12.2 指标

展示：

- 网站用户数。
- 开发者数。
- 访问数。
- 个人作品数。
- 脚本数。
- 博客数。
- 随记数。
- 工作组 / 工作站服务数。
- 日度变化趋势。

### 12.3 约束

- 不展示用户明细。
- 不展示 IP。
- 不展示账号。
- 不展示留言原文。
- 不展示服务提交日志。

## 13. 后台管理设计

### 13.1 布局

后台使用标准管理布局：

- 左侧侧边栏。
- 顶部账号区。
- 主内容区。
- 表格、筛选、编辑抽屉或弹窗。

### 13.2 页面风格

- 以效率为主。
- 少用装饰。
- 操作按钮清晰。
- 危险操作需要确认。
- 管理后台不使用毛笔字体和强叙事动效。

### 13.3 通用状态

后台所有表格页面应包含：

- 加载状态。
- 空状态。
- 错误状态。
- 搜索 / 筛选。
- 分页。
- 删除确认。
- 保存成功提示。
- 保存失败提示。

## 14. 通用交互状态

| 状态 | 要求 |
| --- | --- |
| 加载中 | 使用简洁 loading，不阻塞不相关区域 |
| 空状态 | 明确说明当前没有数据，并提供可操作入口 |
| 错误状态 | 展示可理解错误信息，不暴露系统内部细节 |
| 无权限 | 告知需要登录或没有权限，并提供返回入口 |
| 执行中 | 工作站服务需展示状态卡片和 request_id |
| 成功 | 展示结果摘要和后续入口 |
| 失败 | 展示失败分类、摘要和重新提交入口 |
| 危险操作 | 删除、下架、禁用需二次确认 |

## 15. 可访问性与性能

- 所有核心功能不能依赖动效才能完成。
- 登录、注册、工作站服务提交必须支持键盘操作。
- 输入框和按钮需要清晰 focus 状态。
- 文本和背景对比度要足够。
- 登录动画和小猫交互应支持关闭或降级。
- 大屏看板需要避免长期运行导致浏览器性能下降。
- 移动端优先保证表单可用性。

## 16. 验收要点

- 登录页左右空间约为 1:2。
- 登录页主视觉为黑白光束风格。
- 标题和副标题具备毛笔书写艺术感。
- 登录成功后存在“下坠到向上”的反转动效或降级替代。
- 小猫点击后可进入跟随鼠标状态，且不遮挡表单。
- 主页对访客公开，邀请模块显示锁定状态。
- 工作站以服务卡片展示。
- 日志自动提交服务页按“说明、输入、提交、结果、记录”布局。
- 提交后显示执行中状态卡片并轮询结果。
- 大屏看板为深色全屏数据展示。
- 后台为左侧菜单 + 表格管理布局。
- 所有敏感信息在 UI 中不展示明文。

## 17. 待确认项

- 登录页人物剪影是否使用静态图片、帧动画、CSS/SVG 动画或 Canvas 实现。
- 小猫形象是否需要自定义绘制，还是使用生成式位图资产。
- 毛笔字体是否使用开源字体、图片字形或手写定制字。
- 登录成功反转动画是否只在首次登录播放。
- 移动端是否保留完整品牌视觉，还是简化为表单优先。
- 大屏看板是否需要自动轮播或全屏按钮。
